## 🎯 核心设计亮点

### 1. 架构设计
- **架构模式:** 模块化单体架构 (Modular Monolith)
- **技术栈:** Node.js (NestJS) + PostgreSQL 14+ + Redis 7+
- **部署方式:** Docker 容器化 + Kubernetes 编排
- **架构优势:** 
  - 快速迭代，降低开发复杂度
  - 模块化设计，便于后期拆分微服务
  - 高可用性 99.9%，支持弹性伸缩

### 2. 业务设计
- **会员体系:** 4 级等级 + 积分体系 + 会员卡体系
- **预约规则:** 灵活的规则引擎，支持多种限制条件
- **支付体系:** 微信支付集成，支持多种支付场景
- **消息通知:** 多渠道推送（微信 + 短信），智能频控

### 3. 数据设计
- **核心表:** 8 张表覆盖全部业务场景
- **索引优化:** 合理的索引策略，支持高效查询
- **缓存策略:** Cache-Aside 模式，多级缓存设计
- **数据安全:** 敏感字段加密，完善的权限控制

### 4. 安全设计
- **认证方式:** 微信登录 + JWT Token
- **授权模型:** RBAC 角色权限控制（7 种角色）
- **数据加密:** 传输加密 (TLS 1.3) + 存储加密 (AES-256)
- **安全防护:** XSS/CSRF/SQL注入防护

## 📋 关键业务规则

### 会员等级规则
| 等级 | 积分要求 | 折扣 | 核心特权 |
|------|----------|------|----------|
| 普通会员 | 0 | - | 基础服务 |
| 银卡会员 | 1000 | 95 折 | 专属客服 |
| 金卡会员 | 5000 | 90 折 | 优先预约 |
| 钻石会员 | 20000 | 85 折 | 私教体验 |

### 预约规则
- 最多提前 7 天预约
- 开课前 2 小时可免费取消
- 每日最多预约 2 节课
- 月爽约 3 次加入黑名单

### 积分规则
- 消费 1 元 = 1 积分
- 每日签到 = 1 积分
- 邀请新用户 = 100 积分
- 积分可兑换课程、商品、抵扣现金

## 🔧 技术实现要点

### 数据库优化
- 主键使用 UUID，避免业务信息泄露
- 单表超过 500 万数据考虑分表
- 历史数据定期归档

### 接口设计
- RESTful 风格，统一响应格式
- 写接口必须支持幂等
- 使用唯一请求 ID 防止重复提交

### 性能优化
- 热点数据二级缓存（本地 + Redis）
- 慢查询日志分析，定期优化
- 接口响应时间监控和告警
